אני אהיה חד וברור: **הספאק הישן והחדש לא מתנגשים – הם יושבים על שכבות שונות – אבל כרגע הם גם לא מחוברים מספיק טוב.** אם תטמיע את החדש בלי לייבא עקרונות מהישן, אתה תאבד עומק. אם תנסה "למזג" אחד לאחד – תייצר בלאגן.

### תמונת מצב אמיתית

**הישן** (כפי שאני מכיר ספאקים כאלה):

* Event-first
* פירוק מדויק של אינטראקציות
* Naming קפדני
* הבחנה ברורה בין system / user / product events
* הרבה משמעת, מעט מרקטינג

**החדש**:

* Tool-first (GA4 + Mixpanel)
* גבוה, רחב, לא חד
* מדבר על "לעקוב אחרי הכל" בלי להגדיר מה זה "הכל"
* Governance קיים – וזה טוב
* חסר: מודל אירועים מחייב

### האם הם משלימים אחד את השני?

כן — **אם** אתה עושה את זה נכון.
כרגע הם לא משלימים, הם *מרחפים אחד ליד השני*.

* החדש עונה על **איפה ואיך מודדים**
* הישן עונה על **מה ולמה מודדים**

בלי הישן → דאטה רועש וחסר משמעות
בלי החדש → דאטה יפה בקובץ, לא מחובר למציאות

### ניגודים בעייתיים (קריטי)

1. **"Track all possible data"**
   זה משפט מסוכן.
   GA4 במיוחד *נשבר* מזה. Mixpanel יתנפח ויהפוך לביצה.

2. **אין Event Contract**
   אין:

   * event schema
   * required properties
   * naming convention
   * distinction בין page / action / system

3. **אין Ownership על אירועים עצמם**
   יש ownership על החיבור – אין ownership על הדאטה.

### עם מי כדאי לעבוד?

לא לבחור. **לחלק תפקידים.**

* **GA4**

  * Traffic
  * Acquisition
  * Funnels רחבים
  * SEO / marketing / sessions

* **Mixpanel**

  * Product analytics
  * User journeys
  * Retention
  * Feature usage
  * Everything that matters באמת

אם אתה מודד אותו אירוע בשניהם — זה רק:

* page_view
* session_start
* signup / login
  כל השאר? Mixpanel בלבד.

### האם כדאי למזג ביניהם?

❌ לא למזג
✅ **לכפות שכבת ביניים אחת**

מה כן לעשות (וזה החלק שאתה חייב לבצע, לא "לחשוב עליו"):

#### 1. Analytics Canonical Spec (חובה)

מסמך אחד:

* Event name (canonical)
* Description
* Properties (typed, required / optional)
* Sent to: GA4 / Mixpanel / both
* Trigger location (UI / API / system)

#### 2. Event Taxonomy קשיח

לדוגמה:

* `page_view.*`
* `user.auth.*`
* `course.entered`
* `lesson.completed`
* `chat.message.sent`
  בלי יצירתיות. בלי "נסתדר אחר כך".

#### 3. Mapping Layer

קוד אחד:

* `track(event, payload)`
* בפנים: adapter ל-GA4 / Mixpanel
* אפס קריאות ישירות ל-SDKs בקוד המוצר

#### 4. Governance אמיתי

לא רק "לדבר עם עידו".

* Feature flag per environment
* Dry-run mode
* Debug dashboard
* Event diff בין staging ל-prod

### השורה התחתונה (בלי ליטופים)

* הספאק החדש **לא מספיק**
* הישן כנראה **עמוק יותר מדי בלי קונטקסט מודרני**

אם אתה רציני — השלב הבא הוא שאני מכריח אותך להגדיר:
