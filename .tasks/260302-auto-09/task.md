# Task

## Issue Title

Slow Typing Animation in Welcome Greeting
# 🐞 Bug Report

## 1. Title
 Slow Typing Animation in Welcome Greeting

## 2. Environment
- Environment: dev

## 3. Preconditions
3. Preconditions

User is navigating to the greeting/onboarding flow (e.g., first-time entry or guest session start).

The GreetingFlow component is triggered.

## 4. Steps to Reproduce
4. Steps to Reproduce

Open the application as a new user or in an incognito window.

Navigate to the landing page or onboarding section where the system talks to the student.

Observe the "typing" animation as the system displays the welcome message.

Compare the current speed with a standard human reading/typing pace.

## 5. Expected Result
The typing animation should have a natural, human-like pace. It should be fast enough to keep the user engaged without making them wait too long to read the full message, but slow enough to maintain the "interactive dialogue" feel.

## 6. Actual Result


The animation is significantly slower than a natural typing speed. Users have to wait several seconds for a simple sentence to complete, which creates friction in the onboarding experience and feels laggy.

## 7. Reproducibility
always
