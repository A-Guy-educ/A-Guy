# Task

## Issue Title

Improve Login Page UI Design Consistency
### Description
Update the visual design and layout of the login page to precisely match the provided design reference and align with the existing design system standards.

### Acceptance Criteria
- [ ] The login page visually matches the provided design reference (`https://sample-banana-62009166.figma.site`).
- [ ] All UI elements (buttons, inputs, typography, colors, spacing) utilize established design system tokens and components to ensure global consistency.
- [ ] Input fields and buttons inherit their hover, focus, disabled, and validation error behaviors directly from the standard design system.
- [ ] The headline, supporting text, and internal container text match the exact font families, sizes, weights, and colors specified in the design and design system.
- [ ] Padding, margins, and structural alignment between the header, login container, and background adhere strictly to the design reference and system grid/spacing tokens.
- [ ] Internal UI elements within the login card (input fields, buttons, links) are aligned identically to the design reference.
- [ ] The layout gracefully adapts and maintains visual consistency across viewports, strictly utilizing the standard responsive breakpoints defined by the design system.
