> Audit date: 2026-03-03
> Scope: User-visible flows — UI → Backend → DB → External dependencies
> Framework: Playwright (Chromium)

### Biggest Unblocking Win: Seed Data Fixtures

The single highest-impact investment is **creating reusable test data fixtures** (seeded course → chapter → lesson → exercises of all types). This would:

1. **Un-skip 4 existing E2E specs** (course-selection, exercise-page, v2-conversion-panel, v2-error-display)
2. **Enable 15+ proposed tests** that all need the same base hierarchy
3. **Reduce per-test setup time** from "create everything" to "use fixture"

Recommended: a `tests/e2e/fixtures/seed-data.ts` that creates the full hierarchy via Payload Local API before test suite runs, and cleans up after.
